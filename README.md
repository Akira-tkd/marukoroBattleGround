**作品概要**
作品名:まるころBattleGround
ジャンル:三人称アクションシューティング
制作期間:3ヶ月
制作人数:1人
内容:一気に敵が出現し、その敵を全て倒すとまた次の敵が出現するというウェーブ制になっている三人称シューティングゲーム。ウェーブごとにランダムでプレイヤーを強化できる要素が出現し
自分のプレイスタイルやキャラクターの特徴に合わせて強化し、最終ウェーブまで生存することを目標としている。キャラクターを4種類から選べる点や、ウェーブごとに出現する敵の内容、
強化要素など、毎回状況が同じにはならない。

**制作工程**
1.操作キャラのプレハブ+三人称視点用のカメラ制作
2.インプットシステムのアクションマップを使った操作制御の追加
3.攻撃用のデータをまとめたスクリプタブルオブジェクトの定義
4.攻撃時に発生する追加効果用の基底クラスを定義
5.敵用のクラス構造の設計
6.攻撃の種類に問わず攻撃処理を行えるようにインタフェースを追加
7.ステージの作成
8.ステートマシンによってゲームの状況制御を行えるようにする
9.複数のキャラクターデータを追加+キャラクター選択画面の追加
10.敵キャラクターの追加+出現する敵を複数パターンからランダムで抽選される方式に変更
11.操作説明用のチュートリアルシーンの追加+Mキーによるプレイ中断機能を追加

**アピールポイント**
・汎用性、拡張性を重視してスクリプタブルオブジェクトやインタフェースを利用したので、既存のスクリプトを編集せずに敵や攻撃方法を追加できる
・射出タイプの攻撃の当たり判定にコライダーではなくレイを使用したので弾オブジェクトが高速でも対応可能
・段階的に操作が制限されるチュートリアルの実装

**今後の展望**
・追加しやすいとはいえ、現状敵キャラが3種類しか存在しないのでランダム性との親和性を高めるためにもバリエーションを増やす必要がある
・シングルトンによるグローバルアクセスポイントを複数使っていることに加え、それらのインスタンスが一つしかないことをスクリプト内で保証しているものが少なく、複数人で編集を行うと危険
・BGMやSEが無くプレイしていて状況が分かりづらいので、敵に攻撃が当たった時だけでも効果音があった方が良かった
